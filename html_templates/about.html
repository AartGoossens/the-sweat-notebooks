{% extends "base.html" %} {% set title = "The Sweat Notebooks" %} {% block content %} The Sweat Notebooks is an application that gives you reports about your Strava activities.

I developed this application as a side project because I wanted to try to combine a few technologies:
  • Papermill to parametrize and execute Jupyter Notebook templates.
  • Nbconvert to convert the generated Jupyter Notebooks into html.
  • Strava webhooks to get a webhook trigger from Strava when you upload a new activity.
The rest of the technology stack is:
  • FastAPI, a Python web framework.
  • Docker.
  • Dokku, a Heroku alternative that is running on my personal server.
The design (yes, calling it design might be a stretch) may look very basic. That has a few reasons:
  • I am lazy.
  • I am not a frontend developer and definitely not a designer.
  • I do not want you to get distracted by a fancy user interace.
  • I am a fan of brutalist web design.

Getting started

To get started, head over to the login page and connect with Strava. Then when you upload a new activity to Strava, your reports will be added to the reports page.

Development

This application is still in development. Some features I would like to add:
  • Generating report for activities that were uploaded before a user connected this application to Strava.
  • Support for multiple reports and reports for different types of activities.
  • Allow uploading custom Jupyter Notebook templates.
  • Track features that are computed in the notebook over time by using Scrapbook (another tool that I'd like to try).
